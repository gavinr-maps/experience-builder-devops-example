L’outil Géocoder des localisations à partir d’une table convertit des adresses ou des lieux depuis une table en entrée en coordonnées.

Table en entrée avec les adresses ou lieux à géocoder. L’entrée doit être une table, un fichier  .csv ou un fichier Excel.

Service de localisation à utiliser pour le géocodage. Si le service de localisation que vous souhaitez utiliser ne figure pas dans le menu déroulant, contactez votre administrateur afin qu'il soit configuré pour le géocodage par lots.

Indique si les données en entrée sont contenues dans un ou plusieurs champs.

Champs utilisés par le localisateur pour déterminer les localisations des données. Certains champs sont censés figurer dans les données, comme l’adresse, la ville, l’état et le code postal.

Pays dans lequel se trouvent les données. Si toutes les adresses proviennent d’un même pays, indiquez ce pays pour optimiser la précision des résultats. Sinon, laissez ce paramètre vide.

Spécifie les champs qui seront inclus dans le jeu de données en sortie. Les options sont les suivantes :

  • All (Tous) : tous les champs en sortie du localisateur disponibles seront inclus dans les résultats de géocodage.
  • Minimal : un sous-ensemble de champs du localisateur ainsi que les champs de la table en entrée seront inclus dans les résultats.
  • Location only (Localisation uniquement) : le champ Shape du localisateur ainsi que les champs de la table en entrée seront inclus dans les résultats.

Type de sortie à créer. La sortie peut être un service d’entités, un fichier  .csv ou un fichier .xls.

Nom de l’élément en sortie où les données de localisation seront stockées.

Nom du dossier de My Content (Mon contenu) dans lequel sera enregistré le résultat.
