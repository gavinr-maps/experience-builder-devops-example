Інструмент З’єднання об'єктів додає атрибути з одного шару або таблиці до іншого шару або таблиці на основі просторових або атрибутивних відношень. Як варіант, статистику можна розрахувати, якщо є кілька відповідних записів.

Шар або таблиця, що матиме атрибути із шару з’єднання, приєднані до його/її таблиці.

Шар або таблиця з атрибутами, які будуть приєднані до цільового шару.

Укажіть, чи з’єднувати шари на основі просторового відношення. Для цього необхідно, щоб обидва шари мали геометрію.

Укажіть просторове відношення, яке визначає з’єднання об’єктів. Об’єкти в шарі з’єднання, які мають вказане відношення до цільового шару, будуть з’єднані. Доступні відношення залежать від типу геометрії вхідних даних.

Відстань від об’єктів у цільовому шарі, яка використовуватиметься для визначення необхідності з’єднання об’єктів.

Одиниці вимірювання, які використовуватимуться для визначення відстані.

Укажіть, чи з’єднувати шари на основі відповідних значень полів. Це єдиний варіант, якщо один або кілька вхідних даних є таблицею.

Укажіть атрибутивне відношення, яке визначає з’єднання об’єктів. Об'єкти зіставляються, коли значення полів у шарі з’єднання рівні значенням полів у цільовому шарі.

Укажіть, як з’єднання оброблятимуться, якщо кілька записів з шару з’єднання відповідають критеріям з’єднання. Виберіть із наведених нижче варіантів:

  • З'єднати один з одним— з'єднання одного запису в шарі з’єднання з кожним об’єктом у цільовому шарі.
  • З’єднати один з багатьма—з'єднати всі відповідні об’єкти в шарі з’єднання з цільовим шаром. Результуючий шар буде містити декілька записів про цільовий об'єкт.

Якщо декілька об’єктів у шарі з’єднання відповідають цільовому об’єкту, укажіть, чи потрібно з’єднати один запис із шару з’єднання чи обчислити підсумкову статистику всіх відповідних записів із шару з’єднання.

Якщо існує кілька функцій з’єднання, які відповідають цільовому об'єкту, буде обчислено підсумкову статистику для всіх відповідних записів. Укажіть кількість полів або полів дати з шару з’єднання та статистику, яка обчислюватиметься для кожного поля.

Якщо існує кілька з’єднаних об’єктів, які відповідають цільовому об’єкту, перший запис із таблиці атрибутів шару з’єднання буде включено до вихідних даних. Додатково виберіть поле числа або дати та порядок сортування, який визначатиме порядок таблиці атрибутів.

Визначає, чи будуть невідповідні цільові об'єкти включені до вихідних даних. Внутрішнє з’єднання повертає лише відповідні цільові записи, тоді як Ліве з’єднання повертає всі цільові записи, навіть якщо цільовий об’єкт не має відповідності в шарі з’єднання.

Ім'я шару або таблиці, які буде створено та додано до карти.

Назва папки в Моєму вмісті, де буде збережено результат.
