The Join Features tool adds attributes from one layer or table to another based on specified spatial and attribute relationships. Optionally, statistics can be calculated when there are multiple matching records.

The layer or table that will have attributes from the join layer appended to its attribute table.

The layer or table with the attributes that will be appended to the target layer.

Specify whether to join layers based on a spatial relationship. This requires that both layers have a geometry.

Specify the spatial relationship that determines which features are joined. Features in the join layer that have the specified relationship to the target layer will be joined. The available relationships depend on the geometry type of the inputs.

The distance from the features in the target layer that will be used to determine whether features should be joined.

The units that will be used to specify the distance.

Specify whether to join layers based on matching field values. This is the only option if one or more of the inputs is a table.

Specify the attribute relationship that determines which features are joined. Features are matched when the field values in the join layer are equal to field values in the target layer.

Specify how joins will be handled if multiple records from the join layer meet the join criteria. Choose from the following options:

  • Join one to one— Join a single record in the join layer to each feature in the target layer.
  • Join one to many—Join all the matching features in the join layer to the target layer. The result layer will contain multiple records of the target feature.

When multiple features in the join layer match a target feature, specify whether to join a single record from the join layer or to calculate summary statistics of all matching records from the join layer.

When there are multiple join features that match a target feature, summary statistics for all matching records will be calculated. Specify the number or date fields from the join layer, and the statistics that will be calculated for each field.

When there are multiple join features that match a target feature, the first record from the join layer's attribute table will be included in the output. Optionally, choose a number or date field and the sort order that will determine the ordering of the attribute table.

Determines whether unmatched target features will be included in the output. An Inner join returns only matched target records, whereas a Left join returns all target records, even if the target feature does not have a match in the join layer.

The name of the layer or table that will be created and added to the map.

The name of the folder in My Content where the result will be saved.
