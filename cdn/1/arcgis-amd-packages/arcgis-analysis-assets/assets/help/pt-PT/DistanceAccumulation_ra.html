A ferramenta Acumulação de Distância calcula a distância acumulada para cada célula para introduzir origem, permitindo a distância em linha reta, distância de custo e distância de superfície verdadeira, bem como fatores de custo verticais e horizontais.

A camada de elementos ou raster de entrada que identifica as origens para as quais as distâncias serão calculadas.

Se a entrada for um raster, a mesma tem de consistir em células com valores inteiros válidos (zero é um valor válido) para as origens e às células restantes tem de ser atribuído o valor NoData.

As barreiras que definem os obstáculos para os quais têm de ser criadas rotas à volta.

Para uma camada de raster, as células com um valor (incluindo zero) serão tratadas como uma barreira.

O raster que define os valores de elevação em cada localização da célula.

Os valores são utilizados para calcular a distância real da superfície coberta ao passar entre células.

O raster que define a impedância ou custo para se mover por cada célula.

O valor na localização de cada célula representa a distância de custo-por-unidade para movimentar através da célula. O valor não pode ser negativo ou zero.

Especifica se a distância será calculada com um método planar (terra plana) ou um método geodésico (elipsoide).

O raster que define os valores z para cada localização de célula.

Os valores são utilizados para calcular a inclinação utilizada para identificar o fator vertical incorrido durante o movimento de uma célula para outra.

O fator que define a relação entre o fator de custo vertical e o ângulo de movimento relativo vertical.

Existem várias opções para fatores com modificadores que identificam um gráfico de fator vertical definido. Os gráficos são utilizados para identificar o fator vertical utilizado no cálculo do custo total para mover-se para uma célula vizinha.

O raster que define a direção horizontal em cada célula.

Os valores em cada localização serão utilizados em conjunto com o parâmetro Fator horizontal para determinar o custo horizontal incorrido ao deslocar-se de uma célula para os seus vizinhos.

O fator que define a relação entre o fator de custo horizontal e o ângulo de movimento relativo horizontal.

Existem várias opções para fatores com modificadores que identificam um gráfico de fator horizontal definido. Os gráficos são utilizados para identificar o fator horizontal utilizado no cálculo do custo total para mover-se para uma célula vizinha.

O custo cumulativo inicial com o qual se inicia o cálculo do custo.

O padrão é 0.

A acumulação máxima para o viajante para uma origem.

A acumulação predefinida está na extremidade do raster de saída.

Um multiplicador que será aplicado aos valores de custo.

O padrão é 1.

A direção do viajante ao aplicar os fatores horizontal e vertical.

O nome da camada de raster da distância acumulativa que será criada e adicionada ao mapa. A camada contém a distância acumulativa para cada célula a partir de ou até à origem de menor custo.

O nome da camada de raster da direção de retorno que será criada e adicionada ao mapa. A camada contém a direção em graus.

O nome da camada de raster da direção de origem que será criada e adicionada ao mapa. A camada contém a direção de azimute da célula de origem do menor custo acumulado em graus.

O nome da camada de raster multibanda de localização de origem que será criada e adicionada ao mapa. A camada contém as localizações de origem em que a primeira banda contém um índice de linha e a segunda banda contém um índice de coluna.

O tipo de saída que será criado. As saídas podem ser camadas de imagens em mosaico ou camadas de imagens dinâmicas.

O nome da pasta em O Meu Conteúdo onde os resultados serão guardados.
