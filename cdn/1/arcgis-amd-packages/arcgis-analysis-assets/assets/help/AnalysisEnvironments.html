Environment settings for analysis in Map Viewer.

Specifies the coordinate system for analysis and the result layer.

Specifies the area to be used for analysis.

Adjusts the extent of the output so it matches the cell alignment of the specified snap raster layer.

Specifies the cell size or resolution that will be used for analysis and to create the output raster layer.

Specifies a mask layer in which only the cells that fall within the mask area will be used for analysis.

Specifies how pixel values will be interpolated when the input and output rasters do not align.

Specifies the type of processor to be used. The options are CPU, GPU, and Automatic. If Automatic is specified, the tool will define the processor type.

Specifies the number of raster processing service instances used during the tool run. Supported values include empty strings (the tool will determine the number of raster processing service instances to use), whole numbers (0 or positive integers), and percentages. Percentages must include a percent sign (%) following the percentage value.

Determines whether a warning message will appear if the estimated number of credits surpasses the specified threshold (enabled) or if the analysis will run without displaying a message regardless of the estimated credit consumption (not enabled). The analysis will still honor credit budgeting settings for the organization.

Specifies the threshold at which a warning message will appear based on the estimated credit consumption from running the analysis. The analysis can still be run if the estimate surpasses the threshold.
